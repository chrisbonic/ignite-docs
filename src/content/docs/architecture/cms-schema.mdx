The platform uses the Webflow CMS API with four collections. Each new Webflow site gets new collection IDs and option IDs that must be configured via environment variables.

## Collections Overview

| Collection | Purpose | Key Fields |
|-----------|---------|------------|
| **Clients** | Prospect company records | name, token, status, source-of-truth, quote-bank |
| **Ideas** | 7-slide pitch pages (40+ fields) | hero lines, quotes, gap cards, idea visual, scenarios, steps |
| **Pages** | Sales, partner co-sell, and package pages | hero, proof, problem, approach, CTA, partner, package fields |
| **Partners** | Shared partner organizations | name, badge, logo, categories, active, featured |

## Clients Collection

| Field | Type | Description |
|-------|------|-------------|
| `name` | PlainText | Company display name |
| `token` | PlainText | Unguessable URL token (e.g., `wm-7f3a9b`) |
| `tagline` | PlainText | Portfolio tagline |
| `description` | PlainText | Company about/mission |
| `nav-context` | PlainText | Nav bar label |
| `company-url` | PlainText | Company website URL |
| `book-call-url` | Link | Booking page URL |
| `logo-url` | PlainText | Client logo image URL |
| `status` | Option | `active` / `archived` / `draft` |
| `source-of-truth` | RichText | Perplexity research document |
| `quote-bank` | RichText | Verified exec quotes with sources |
| `presentation-context` | PlainText | Meeting context, audience, constraints |
| `partner-1..4-name/desc/badge` | PlainText | Partner display fields (4 partner slots) |

## Ideas Collection

46 fields organized by slide. Each idea references a Client.

### Meta Fields
| Field | Type | Description |
|-------|------|-------------|
| `client` | Reference | Link to Clients collection |
| `number` | Number | Display order |
| `category` | PlainText | Category label (e.g., "Revenue & Growth") |
| `category-color` | Option | `indigo` / `purple` / `rose` / `blue` |
| `audience-flavor` | Option | Target audience lens |
| `archived` | Switch | Hide from public view |

### Slide Fields (per slide)
| Slide | Fields |
|-------|--------|
| **1. Hero** | `hero-line-1`, `hero-sub`, `hero-image-url`, `video-url` |
| **2. Quote** | `quote-label`, `quote-text`, `quote-attribution`, `quote-source` |
| **3. Gap** | `gap-line-1/2/3`, `gap-card-1..3-title/desc`, `gap-evidence-url/label` |
| **4. Idea** | `idea-line-1/2`, `idea-description`, `idea-visual-html`, `idea-image-url/caption` |
| **5. Scenarios** | `scenarios-headline`, `scenario-1/2-tag/title/desc` |
| **6. Delivery** | `step-1..3-title/desc` (Step 4 is hardcoded) |

## Option IDs

Webflow CMS option fields use hash IDs. Each Webflow site generates unique option IDs. These are resolved at runtime via `webflow-options.ts` - no hardcoded option maps in the codebase. The same code works across test and production sites without changes.

When writing to option fields via the API, use the **name string** (e.g., `"active"`) not the hash ID. The shared library handles the translation.
