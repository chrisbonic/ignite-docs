## What shipped

### 1. Astro 6 → 5.18 downgrade (admin + site)

Webflow support confirmed on 2026-04-15 that Astro 6.x is not supported and has no ETA. Both deploy targets unblocked by downgrading:

- `admin/package.json`: `astro: ^6.0.5` → `^5.18.0`
- `site/package.json`: `astro: ^6.0.5` → `^5.18.0`
- `@astrojs/cloudflare@^12.6.13` kept as-is (works with both majors)
- `@astrojs/react@^5.0.3` resolves fine under Astro 5 (site only)

**Zero code changes required.** Our API surface — routes, middleware, Cloudflare adapter, `security.checkOrigin` — is stable across both majors. Local `npm run build` passed first try in both repos.

### 2. React deps re-added to admin

First post-downgrade admin deploy failed with:

```text
Cannot find package 'react-dom' imported from /repo/node_modules/@astrojs/react/dist/version.js
```

**Root cause:** Webflow Cloud builder v1.2.0 auto-installs `@astrojs/react@^4.0.0` as a framework requirement ("Required for Webflow React components"). Session 27 cleanup (commit `ac438e8`) had removed `react`/`react-dom` from admin because admin uses no React components. The builder's auto-install then had no peer to resolve.

**Fix:** `react@^19.2.4` + `react-dom@^19.2.4` re-added to admin. Site already had them.

## How to apply

If you are reading this:

- **Do not bump Astro to 6.x in admin or site.** Webflow does not support it.
- **Do not remove `react` or `react-dom` from admin or site `package.json`.** Webflow Cloud's builder auto-installs `@astrojs/react@^4` and requires the peer deps.
- **CI is advisory.** A red ✗ does not block anything. Look at it, decide if it matters, move on.
- **The Webflow feedback memo at `docs/webflow-cloud-astro-6-feedback.md` is not yet sent.** Tune the tone before sharing externally.
- **Pre-existing type errors are known.** `npm run typecheck` in admin/site will show them; they were there before this session. Cleaning them up is Session 32 material.
