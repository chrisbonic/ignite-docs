Sales pages are problem-focused landing pages that frame a specific pain point and position Ignite's approach as the solution. Unlike idea pitches (which showcase a specific innovation concept), sales pages target a broader business problem.

## When to Use Sales Pages

- A prospect has a specific pain point you want to address directly
- You need a landing page for a particular meeting or campaign
- You want to complement idea pitches with a broader strategic narrative

## Generation Flow

1. From the client hub, go to **Generate Sales**
2. The system first identifies **sales angles** from the client's research
   - Each angle is a specific problem framing (e.g., "The Trust & Safety Gap", "Revenue Leakage")
   - Claude analyzes the source of truth to find the strongest problem narratives
3. Review the angles - approve, reject, or edit
4. Generate full sales pages from approved angles

## Sales Page Structure

Each sales page contains:

| Section      | Content                                                       |
| ------------ | ------------------------------------------------------------- |
| **Hero**     | Bold pain statement headline + supporting context             |
| **Proof**    | Social proof - logos, quotes, credibility markers             |
| **Problem**  | Evidence-based problem framing using their own words and data |
| **Approach** | How Ignite addresses this - visual diagram + step cards       |
| **Media**    | Supporting images (up to 3)                                   |
| **CTA**      | Urgency headline + booking button                             |

## Sales Pages in Packages

Sales pages can be added to packages alongside ideas. They work well as the opening page of a package - frame the problem, then show the ideas that solve it.

All sales pages serve from the same URL pattern: `/c/[token]/p/[slug]`.
